API CONTRACT (for Phase 2 integration):

Your real API must return these shapes:

Sport {
  id: string
  name: string
  slug: string
  icon: string
  group: 'us' | 'intl' | 'combat'
}

League {
  id: string
  sportId: string
  name: string
  country: string
  season: string
}

Team {
  id: string
  leagueId: string
  name: string
  abbreviation: string
  city: string
}

Game {
  id: string
  sportId: string
  leagueId: string
  homeTeam: Team
  awayTeam: Team
  startTime: ISO-8601 string
  status: 'scheduled' | 'live' | 'halftime' | 'final' | 'postponed' | 'canceled'
  clock: string | null          (e.g., "5:22")
  period: string | null         (e.g., "Q4", "2nd", "HT")
  homeScore: number | null
  awayScore: number | null
  venue: string
  hasOdds: boolean
}

OddsMarket {
  gameId: string
  sportsbookId: string
  marketType: 'moneyline' | 'spread' | 'total'
  selection: 'home' | 'away' | 'over' | 'under'
  line: number | null
  oddsAmerican: number
  updatedAt: ISO-8601 string
}

Functions to implement in mockData.js:
  - getSports() → Sport[]
  - getLeagues(sportId?) → League[]
  - getGames({ sport, dayOffset, status, league }) → Game[]
  - getGameById(gameId) → Game | null
  - getOddsForGame(gameId) → OddsMarket[]

See mockData.js for full docstrings.
